# new_pipeline_bot

텔레그램 기반 **뉴스 키워드 알림 + 시나리오 분석 + 시장 브리핑** 통합 봇입니다.  
한국 및 글로벌 금융 뉴스를 자동으로 수집하고, 사용자가 등록한 키워드에 맞춰 알림을 보내며, AI를 이용해 뉴스 흐름을 시나리오(스토리라인)로 구조화해 보여줍니다.

---

## 주요 기능

- **뉴스 키워드 알림**
  - RSS 기반으로 금융 관련 뉴스를 자동 수집
  - 사용자별 관심 키워드 등록
  - 새 기사 중 키워드가 매칭되면 텔레그램으로 즉시 전송

- **AI 시나리오 파이프라인**
  - 새 기사들을 Google Gemini 기반 시나리오 빌더에 투입
  - 시나리오 신규 생성 / 기존 시나리오에 이벤트 노드 추가
  - 시나리오별 타임라인(이벤트 흐름)을 텔레그램에서 조회 가능
  - `/rag` 명령어로 RAG 기반 AI 분석 제공

- **시장 스케줄 & 브리핑**
  - 한국 증시 기준 개장/마감 알람
  - 네이버 증권 데이터를 활용한 테마/업종 급등 브리핑
  - 장초반(10:00), 장마감 후(16:00) 등 시점별 브리핑 자동 전송

- **사용자/뉴스/시나리오 DB 관리**
  - SQLite 기반 뉴스 DB, 시나리오 DB 관리
  - 사용자, 키워드, 전송 내역, 시나리오·노드 통계 제공

## 명령어 정리

### 기본 명령어

- `/start`  
  봇 시작 및 기본 메뉴 표시, 사용자 등록

- `/help`  
  전체 명령어와 사용법 안내

- `/add <키워드>`  
  알림 받을 키워드 추가  
  예) `/add 반도체`, `/add 미국 금리`

- `/remove <키워드>`  
  등록된 키워드 삭제

- `/keywords`  
  내가 등록한 키워드 목록 확인

- `/search <키워드>`  
  수집된 뉴스 DB에서 해당 키워드 포함 기사 검색  
  예) `/search 금리`

- `/status`  
  봇 전체 통계 출력  
  - 수집된 뉴스 개수  
  - 등록 사용자 수  
  - 전체 키워드 수  
  - 누적 전송 건수  
  - 시나리오/노드/카테고리 통계

- `/fetch [veryfast|full]` (관리자 전용)  
  수동으로 뉴스 수집 트리거  
  - `veryfast`: 피드당 10개, 빠른 테스트용  
  - `full`: 피드당 40개, 저녁 대량 수집과 동일

### 시나리오 관련 명령어

- `/scenarios [카테고리]`  
  시나리오 목록 조회  
  - 카테고리 예: `energy`, `finance`, `geopolitics`, `tech`, `trade`, `macro`, `corporate`  
  - 인자를 생략하면 최신 시나리오 목록 반환

- `/scenario <id>`  
  특정 시나리오 상세 보기  
  - 설명, 카테고리, 키워드  
  - 최근 이벤트 노드 타임라인 (최대 15개)

- `/rag <질문>`  
  RAG 기반 AI 시나리오 분석  
  - 입력 질문과 관련된 시나리오들을 검색 후,  
    Gemini로 종합 분석 결과 생성  
  - 예) `/rag 미-중 무역전쟁 현황`,  
    `/rag 삼성전자 반도체 관련 시나리오`

### 시장 브리핑 명령어

- `/briefing`  
  현재 시점 기준 테마/업종 급등 브리핑 요청  
  - 네이버 증권 데이터를 수집 후,  
    테마/업종 +3% 이상 급등 구간 중심으로 요약

## 자동 스케줄

봇은 `python-telegram-bot`의 Job Queue를 사용해 다음 작업을 자동 실행합니다.

- **뉴스 수집 및 시나리오 파이프라인**
  - 매시간: `veryfast` 모드 (피드당 10개)
  - 매일 22:00 KST: `full` 모드 (피드당 40개)
  - 각 수집 사이클마다
    - 신규 기사 필터링
    - 사용자 키워드 알림 전송
    - 시나리오 파이프라인 실행 (신규/업데이트)
    - 새 시나리오 및 주요 업데이트 시 전체 브로드캐스트

- **장 알람 & 브리핑 (KST 기준)**
  - 09:00: 장 개장 알림
  - 10:00: 장초반 테마/업종 +3% 브리핑
  - 15:30: 장 마감 알림
  - 16:00: 장마감 후 테마/업종 +3% 브리핑

## 아키텍처 개요

- **봇 엔트리포인트**
  - `bot.py`
  - 텔레그램 명령어 핸들러, 콜백 핸들러, 에러 핸들러
  - Job Queue를 이용한 스케줄 작업 등록

- **데이터 계층**
  - `db_manager.py`  
    - 사용자, 키워드, 뉴스, 전송 기록 등 관리  
    - 신규 기사 필터링, 검색, 통계 조회
  - `scenario_db.py`  
    - 시나리오 및 노드(이벤트) 관리  
    - 카테고리별/전체 조회, 통계

- **뉴스 수집 & 분석**
  - `news_fetcher.py`  
    - RSS 피드에서 뉴스 수집  
    - 키워드 기반 필터링
  - `scenario_builder.py`  
    - Gemini API를 이용해 기사 → 시나리오 변환  
    - 시나리오 생성/업데이트, RAG 쿼리 처리

- **시장 브리핑**
  - `naver_market_briefing.py`  
    - 네이버 증권 테마/업종 데이터 수집  
    - 브리핑용 메시지 문자열 생성

- **기타**
  - `api.py`: 외부 API 유틸리티
  - `reset_scenario_db.py`: 시나리오 DB 초기화/리셋 스크립트
  - `.env.template`: 환경 변수 예시 템플릿

## 설치 및 실행 방법

### 1. 리포지토리 클론

```bash
git clone https://github.com/Gachon-junghyun/new_pipeline_bot.git
cd new_pipeline_bot
```

### 2. 가상환경 및 의존성 설치

```bash
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

### 3. 환경 변수 설정

`.env.template`를 참고해 `.env` 파일을 생성합니다.

```bash
cp .env.template .env
```

`.env` 파일에 아래 값을 채워 넣습니다.

- `BOT_TOKEN`  
  텔레그램 BotFather에서 발급받은 봇 토큰

- `ADMIN_USER_ID`  
  관리자 텔레그램 user id (정수)

- `GEMINI_API_KEY`  
  Google Gemini API 키

- (선택) `SIMILARITY_THRESHOLD`  
  시나리오 매칭 임계값 (기본 65)

- (선택) `MAX_PROCESS_PER_RUN`  
  한 사이클에서 처리할 최대 기사 수 (기본 30)

### 4. DB 초기화

처음 실행 시, `bot.py`에서 자동으로 DB 초기화가 수행됩니다.  
필요하면 `reset_scenario_db.py`로 시나리오 DB를 리셋할 수 있습니다.

### 5. 봇 실행

```bash
python bot.py
```

로컬에서 실행 후, 텔레그램에서 봇을 찾아 `/start`를 입력하면 됩니다.

## 개발 메모

- Python 3.10+ 권장
- `python-telegram-bot`의 비동기 스타일(`Application`, `async def` 핸들러) 사용
- 시나리오 파이프라인은 CPU 작업이므로 `asyncio.get_event_loop().run_in_executor()`로 분리
- 텔레그램 메시지 길이 제한(4096자)을 고려해 RAG 응답 및 시나리오 텍스트를 자릅니다.
